# 版本管理说明

## 版本显示

应用会在页面底部以小字体显示当前版本号，格式为：`v1.1.0`

这样你可以快速确认是否在使用最新部署的版本。

## 如何更新版本号

每次部署新版本前，请更新 `package.json` 中的版本号：

### 版本号格式

使用语义化版本号（Semantic Versioning）：
- **主版本号.次版本号.修订号** (例如：1.1.0)
- **主版本号**：重大更新，不兼容的 API 修改
- **次版本号**：新功能，向后兼容
- **修订号**：bug 修复，向后兼容

### 更新方法

#### 方法一：手动编辑 package.json

```json
{
  "version": "1.1.0"  // 改为新版本号，如 "1.2.0"
}
```

#### 方法二：使用 npm version 命令（推荐）

```bash
# 修订号 +1 (1.1.0 -> 1.1.1)
npm version patch

# 次版本号 +1 (1.1.0 -> 1.2.0)
npm version minor

# 主版本号 +1 (1.1.0 -> 2.0.0)
npm version major
```

这些命令会自动：
- 更新 `package.json` 中的版本号
- 创建一个 git commit
- 创建一个 git tag

### 部署流程

1. **更新版本号**：
   ```bash
   npm version patch  # 或其他版本类型
   ```

2. **提交更改**：
   ```bash
   git push origin main
   git push --tags  # 推送版本标签（可选）
   ```

3. **部署应用**

4. **验证版本**：
   - 访问应用
   - 查看页面底部是否显示新版本号

## 技术实现

- 版本号存储在 `package.json` 的 `version` 字段
- 后端 API: `GET /api/version` 返回版本信息
- 前端在页面加载时自动获取并显示版本号
- 版本号显示在页面底部，使用小字体和等宽字体
